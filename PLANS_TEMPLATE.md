# Plan Template for Larger Codex Tasks

Use this when a task feels too large for a single coding pass.

## Context
What part of the repo is changing, and why?

## Goal
What is the exact outcome?

## Non-goals
What should explicitly *not* be changed?

## Files to inspect first
List the files Codex should read before editing.

## Proposed implementation plan
Break the task into 3–7 concrete steps.

## Risks / unknowns
What could go wrong, or what still needs a choice?

## Verification
What commands or checks should be run?

## Output contract
What should the final answer from Codex include?
